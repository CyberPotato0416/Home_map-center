# 🗺️ 591Premium 雙北都市 GIS 智慧租屋地圖系統 (Rent GIS Map Master Plan)

本專案旨在解決 **「591 租屋連結因每日重刊而失效，難以記錄比較」** 的痛點，並結合 **都市 GIS 資料（行政區界、租金統計、捷運與公車路網）**，打造一個以 **民權東路三段辦公室（公司中心點）** 為核心的 **591Premium** 客製化地圖式租屋紀錄與決策分析系統。


---

## 🏗️ 系統網頁架構與技術棧 (Web Architecture & Tech Stack)

為了讓 `AI.dev` 開發流暢且免於複雜的環境建置，我們採用**輕量、高效、無伺服器 (Serverless)** 的前端 GIS 架構：

- **前端骨架 (Structure)**: HTML5 Semantic Tags
- **視覺樣式 (Styling)**: Vanilla CSS (以變數管理配色，採用深色模式、玻璃摩砂質感與微交互動畫，打造高科技感儀表板)
- **地圖引擎 (Map Engine)**: **Leaflet.js** (開源免 Key、效能優秀，搭配 OpenStreetMap 圖資)
- **資料儲存 (Data Storage)**:
    - **運行時**: 瀏覽器 `localStorage` (實現隨加隨存、免資料庫)
    - **永久存檔**: JSON 匯入/匯出 (備份 591 租屋點，連結失效也沒關係)
- **GIS 圖資來源**:
    - **雙北行政區界**: 簡化版 雙北市 GeoJSON 邊界資料
    - **捷運線與站點**: 捷運路網坐標資料 (結合 Leaflet Polyline/Marker)

## 🤝 雙端 AI 混合開發協同規範 (Hybrid Dev Collaboration Model)

本專案採用 **「地端 AI (Antigravity)」** 與 **「雲端 AI (AI.dev)」** 的雙端混合開發模式，為確保兩者協作時不會產生程式碼衝突或檔案遺失，請遵循以下分工與同步規則：

1. **分工邊界 (Responsibilities)**:
   - **雲端 AI (AI.dev)**: 專注於 `src/` 目錄下的**前端網頁功能、UI 元件與評分邏輯**開發。
   - **地端 AI (Antigravity)**: 專注於非網頁程式碼的處理，包括 **`.md` 規格與說明文件撰寫**、**`.csv` 資料整理（如 `public/rentals_import.csv` 的補充修訂）** 以及 **`.py` 地端工具**等描述或輔助性檔案。

2. **儲存庫檔案與 Git 同步規則 (Git Synchronization & Ignore Rules)**:
   - **專案檔案保護**: 以下檔案屬本地或開發資產，嚴禁雲端/地端 AI 進行清理或刪除：
     - `extract_591_to_csv.py` (本地爬蟲工具)
     - `public/rentals_import.csv` (租屋匯入資料)
     - `public/rentals_images/` (本地相片目錄)
     - `rent_map_specs/` (開發規格書目錄)
   - **相片直接儲存倉庫**: 為了讓 `AI.dev` 與前端可以直接取得已下載的物件相片（例如 `gov_112` 或 591 下載相片），`public/rentals_images/` **已解除 `.gitignore` 忽略** 並正式納入 Git 倉庫管理。雙端 AI 開發時請直接存取倉庫中的圖片路徑，不再需要另外向 591 請求抓圖。

## ⚖️ 租屋物件加權評分與決策矩陣 (Weighted Scoring Matrix)

本系統之決策分數（預設為 100 分基準，在 **Phase 5** 進行計算）將結合「GIS 通勤距離」與「Ragic 篩選加權屬性」進行綜合評分。評分標準區分為 **布林指標（是/否/未知）** 與 **漸變/量化指標**：

### 1. 布林指標 (Boolean Metrics)
此類指標採二值化加扣分：
*   **電梯 (`電梯` / `有電梯`)**：
    *   `是` (有) ➡️ +15 分。
    *   `否` (無) ➡️ 0 分（若無電梯且樓層高，則觸發下方樓層扣分機制）。
*   **獨立洗衣機 (`洗衣機` / `獨洗`)**：
    *   `是` ➡️ +10 分。
    *   `否` (共用或無) ➡️ -5 分。
*   **變頻冷氣 (`變頻冷氣`)**：
    *   `是` ➡️ +10 分。
    *   `否` (定頻或無) ➡️ -5 分。
    *   `未知` (或黃色燈號) ➡️ 0 分。
*   **垃圾代收 (`垃圾代收`)**：
    *   `是` ➡️ +15 分。
    *   `否` ➡️ 0 分。
*   **租屋補助 (`租屋補助`)**：
    *   `是` (可申請) ➡️ +20 分。
    *   `否` (不可申請) ➡️ 0 分。
*   **有陽台 (`陽台`)**：
    *   `是` ➡️ +8 分 (供模型噴漆與透氣使用)。
*   **有公共空間/頂樓 (`公共空間` / `頂樓`)**：
    *   `是` ➡️ +12 分 (擺放雜物、露台或擺放模型器材等)。
*   **有廚房 (`廚房` / `開伙`)**：
    *   `是` ➡️ +10 分。

### 2. 漸變與量化指標 (Gradient / Categorical Metrics)
此類指標依數值高低或類別等級給予漸進式分數：
*   **裝潢與衛浴星等 (`裝潢等級`, `衛浴等級`)**：
    *   評分採 `1 - 5` 星級漸變：`星等分數 = (等級 - 3) * 5`。
    *   5 星 ➡️ +10 分（高奢極簡、全新裝修）
    *   4 星 ➡️ +5 分
    *   3 星 ➡️ 0 分（一般油漆、堪用）
    *   2 星 ➡️ -5 分
    *   1 星 ➡️ -10 分（破舊老屋）
*   **電費計價 (`電費` / `度`)**：
    *   依每度電價進行漸變扣分：
    *   $\le 4.0$ 元/度 ➡️ +5 分（台電計價或良心價）
    *   $4.1 - 5.0$ 元/度 ➡️ 0 分
    *   $5.1 - 6.0$ 元/度 ➡️ -5 分
    *   $\ge 6.1$ 元/度 ➡️ -15 分
*   **電表類型 (`電表類型`)**：
    *   `獨立電表` ➡️ +10 分。
    *   `共用電表` ➡️ -5 分.
    *   `未知` ➡️ 0 分。
*   **樓層與無電梯懲罰 (`樓層` 與 `電梯` 交叉判定)**：
    *   若有電梯：樓層高低不扣分，頂加或地下室除外。
    *   若無電梯：依樓層高低進行**階梯式扣分**：
        *   1 - 2 樓 ➡️ 0 分
        *   3 - 5 樓 ➡️ -5 分 (大學住 5 樓耐受經驗)
        *   $\ge 6$ 樓 ➡️ -10 分
        *   B1 (地下室) ➡️ 一律 -30 分
*   **停車與停車費 (`停車位`, `停車費`)**：
    *   `室內停車位`且`停車費 = $0` ➡️ +10 分。
    *   `無停車位`或`未知` ➡️ 0 分。

## 📅 分步開發計畫 (Phased Roadmap)

為了讓您的 `AI.dev` 能高效、穩定地逐步開發，我們規劃了 **6 個獨立的開發階段**。您可以將後續產出的每個 `.md` 規格書，按順序依次餵給 `AI.dev`：

*   **Phase 1: 基礎地圖與核心標記 (Base Map & Company Center)**
    *   引入 Leaflet.js，建立以公司中心點為中心的地圖，標記 25km 範圍圈與公司位置。
*   **Phase 2: 行政區劃與租金統計底圖 (District Heatmap & Rental Levels)**
    *   匯入雙北 GeoJSON 行政區，並利用政府租金統計值著色，呈現租金區間分級的漸層熱圖效果。
*   **Phase 3: 捷運系統與主要公車路線 (Transit GIS Layer)**
    *   在地圖上繪製紅線、藍線、文湖線等捷運路線與重要站點，增加通勤便利性視覺參考。
*   **Phase 4: CSV 租屋資料導入與唯讀地圖標記 (CSV Import & Read-Only Pins)**
    *   提供 Excel 租屋 CSV 檔導入與動態表頭解析，在地圖上渲染唯讀租金價格徽章，並儲存於 `localStorage`。
*   **Phase 5: 591 風格資訊面板與 GIS 通勤分析 (Details Panel & Commute Scoring)**
    *   製作精美的側邊資訊展示欄，自動計算點位與公司之直線距離及最近捷運站距離，產出通勤分數。
*   **Phase 6: 資料匯出與進階篩選 (CSV Export & Advanced Filtering)**
    *   提供 CSV 匯出備份功能，以及預算/坪數/通勤時間的交叉篩選，完善決策功能。
