# 📋 Phase 4: CSV 租屋資料導入與唯讀地圖標記 (CSV Import & Read-Only Pins)

## 🎯 開發目標
本系統定位為純靜態、輕量化決策工具，因此**不在前端網頁中提供單筆新增、編輯、刪除的繁瑣表單彈窗**。
使用者會直接在 Excel 或 Google 試算表中整理好 591 收集來的租屋資料，並將其匯出為 CSV 檔。網頁端僅提供 **「CSV 檔案導入」** 功能，導入後解析為 JSON 儲存於 `localStorage`，並在地圖上繪製唯讀標記。若需要修正資料，使用者只需修改本機的 CSV 檔並重新導入覆蓋即可。

## 🎨 視覺與交互設計
1.  **地圖價格標記 (Custom Price Markers)**:
    *   不要使用 Leaflet 預設的藍色水滴標記，請改用 `L.divIcon` 繪製自訂 HTML 標記。
    *   標記外觀：一個深色微型圓角卡片，中間直接顯示**租金縮寫** (例如：`$18.5K` 或 `$12K`)，並有發光邊框。
2.  **唯讀互動機制 (Read-Only Popup & Sidebar Linking)**:
    *   地圖上沒有右鍵或雙擊地圖新增點位的事件。
    *   **極簡唯讀地圖氣泡框 (Minimal Popup)**：點擊地圖價格標記時，彈出的 Leaflet Popup 僅保留極少量的最關鍵欄位：**「物件名稱」、「月租金」與「591 原始網頁連結」**。不放任何照片或次要屬性，亦不提供編輯或刪除按鈕，保持地圖視野整潔。
    *   **側邊欄自動連動**：在點擊價格標記的同時，系統必須自動展開右側邊欄（若原先為收合狀態），並在側邊欄完整呈現該物件的所有詳細資訊（如相片輪播、通勤分數與 Excel 導入的所有自訂詳細欄位）。
3.  **導入 UI 元件**:
    *   在右側面板的 **TAB 4**，設計一個精美的 **「📥 匯入租屋 CSV 檔」** 按鈕（玻璃摩砂質感與懸停動畫）。

## 📊 彈性欄位與動態表頭規範 (Flexible CSV Schema & Dynamic Headers)
由於未來 Excel 範本的欄位數與表頭名稱可能增減變更（例如在 Phase 7/8 進行微調），本系統的 CSV 解析器**絕對不可寫死列索引或嚴格限制欄位名稱**。必須設計「活動表頭（動態對應與模糊匹配）」機制：

### 1. 核心欄位模糊匹配 (Required Core Columns Mapping)
解析器在讀取首列表頭時，應使用以下關鍵字同義詞來動態提取核心 GIS 與地圖標記屬性：
*   **緯度座標 (`lat`)**: 匹配包含 `lat`、`latitude`、`緯度` 關鍵字的表頭。
*   **經度座標 (`lng`)**: 匹配包含 `lng`、`longitude`、`經度` 關鍵字的表頭。
*   **月租金 (`price`)**: 匹配包含 `price`、`rent`、`租金`、`價格` 關鍵字的表頭。
*   **物件標題 (`title`)**: 匹配包含 `title`、`name`、`名稱`、`標題`、`租屋` 關鍵字的表頭。

### 2. 陣列/媒體特殊解析
*   **相片網址 (`images`)**: 若表頭包含 `image`、`photo`、`照片`、`圖片`，其值若含有分號 `;` 或半形逗號 `,`，則自動使用 `.split()` 分割成相片網址陣列。
*   **優選標籤 (`pros`)** 與 **缺點標籤 (`cons`)**: 若表頭包含 `pros`、`優點`、`cons`、`缺點`、`標籤`，以相同方式切割為標籤陣列。

### 3. 動態自訂欄位 (Dynamic Custom Fields)
*   對於核心與媒體之外的其他任何表頭（例如未來使用者在 Excel 自訂的 `坪數`、`樓層`、`房型`、`押金`、`寵物規定` 等），解析器必須**動態讀取表頭名稱作為 Key，單元格內容作為 Value**，以完整的 `JSON` 屬性儲存在點位物件中。這能確保未來不論 Excel 範本如何擴充欄位，網頁端都無需重寫程式碼。

## 💾 資料儲存與 Local Storage 操作
*   **讀取**: 網頁載入時，檢查 `localStorage.getItem('my_rental_pins')`。若有資料，則解析成 JSON 並逐一渲染成地圖價格標記。
*   **寫入**: 當使用者成功匯入新的 CSV 檔案時，解析完成後直接**覆蓋寫入** `localStorage.setItem('my_rental_pins', JSON.stringify(rentals))`，並清除地圖舊點、重新渲染新點位。

## 🛠️ 實作步驟指引

### 1. 建立 CSV 上傳與動態解析邏輯
*   點擊「匯入租屋 CSV 檔」時，觸發隱藏的 `<input type="file" accept=".csv">`。
*   使用 `FileReader` 讀取 CSV 內容，並使用簡單的分隔符號解析邏輯（或引入輕量級 `PapaParse` 函式庫）來解析 CSV。
*   **動態表頭處理**：首行讀取為欄位鍵值陣列 `headers = rows[0]`。遍歷後續每一行 `row` 時，動態組合出物件，並使用模糊匹配規則找出 `lat`, `lng`, `price`, `title` 儲存，其餘欄位以 `rowObject[headerName] = value` 形式全數保留儲存。

### 2. 使用 `L.divIcon` 繪製租金標記
*   取得租金數值後，計算顯示文字（例如：`price = 18500` ➡️ `$18.5K`）。
*   套用自訂樣式：
    ```javascript
    const priceText = `$${(rental.price / 1000).toFixed(1)}K`;
    const myIcon = L.divIcon({
      className: 'custom-rent-marker',
      html: `<div class="marker-badge">${priceText}</div>`,
      iconSize: [50, 25]
    });
    L.marker([rental.lat, rental.lng], {icon: myIcon}).addTo(map);
    ```
